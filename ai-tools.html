🔧 AI 工具的搭建原理（三层架构）

① 训练大模型（底层）

用海量文本（书籍、网页、代码）对神经网络进行预训练，使其学会语言规律。
类比：一个读过互联网所有书籍的"超级学生"。
关键技术：Transformer 架构 + RLHF 人类反馈强化学习。
成本：训练一次需要数千万~数亿美元。

② API 开放（中间层）

厂商将大模型封装为 API，按调用次数/Token 收费。
开发者无需自己训练模型，调用 API 即可获得 AI 能力。
类比：水电公司发电 → 你家插上插头就能用电。
典型价格：约 ¥0.01~0.1 / 千 Token（1 Token ≈ 0.75 个英文单词）。

③ 产品包装（应用层）

在 API 基础上添加 UI 界面、记忆系统、工具调用、文件上传等功能。
ChatGPT = GPT API + 对话界面 + 插件系统 + 联网搜索
Claude Code = Claude API + 终端环境 + 文件读写 + Git 操作
Trae = 豆包 API + VS Code 编辑器 + 项目上下文感知
Copilot = GPT/Claude API + IDE 插件 + 代码补全逻辑

④ Agent 自主代理（最新趋势）

让 AI 不只是回答问题，而是自主规划、调用工具、执行多步骤任务。
例如 Claude Code：你说"帮我重构这个项目"，它会：
1. 读取所有文件 → 2. 分析代码结构 → 3. 制定重构计划 → 4. 逐个修改文件 → 5. 运行测试验证
这就是 AI Agent — 从"工具"进化为"助手"。
